# Tennis Sunlete 대진 생성 스펙

## 1. 문서 목적

- 본 문서는 선레테 서비스의 주간 대진 생성 규칙을 정의한다.
- 대상 기능은 마스터 페이지에서 수행하는 대진 생성 및 확정 기능이다.
- 현재 구현을 기준으로 정리하되, 이후 서버 함수 또는 별도 대진 엔진으로 이전할 수 있도록 정책과 구현 세부를 구분한다.

## 2. 적용 범위

- 대상 페이지: `src/app/pages/MasterPage.tsx`
- 대상 데이터: `schedules`, `attendance_requests`, `doubles_matches`, `users`
- 대상 사용자: 마스터 계정만 대진 생성 및 확정 가능

## 3. 용어 정의

- **참석자(participants)**: 출석 신청자 중 실제 대진 생성 대상이 되는 사용자 목록
- **대기자(waitlist)**: 참석 신청은 했지만 최대 정원 초과로 대기 상태인 사용자 목록
- **대진 생성(generate draw)**: 메모리/UI 상에서 6경기 구성을 계산하는 행위
- **대진 확정(confirm bracket)**: 생성된 6경기를 `doubles_matches`에 저장하는 행위
- **혼복(mixed)**: 남성/여성 조합을 우선하는 복식 생성 모드

## 4. 생성 대상 일정 조건

대진 생성은 아래 조건을 모두 만족할 때만 허용한다.

1. 마스터가 특정 일정을 선택한 상태여야 한다.
2. 일정 상태가 `draw_waiting` 이어야 한다.
3. 참석자 수가 최소 6명 이상이어야 한다.

생성 불가 시 UI는 아래와 같은 수준의 오류를 제공해야 한다.

- 일정 미선택: 경기 일정을 선택하라는 메시지 표시
- 상태 불일치: 일정 마감 후 대진표 생성 대기중 상태에서만 생성 가능 메시지 표시
- 인원 부족: 6경기 생성을 위해 최소 6명의 참석자가 필요하다는 메시지 표시

## 5. 참석자 선정 규칙

참석자는 대진 생성 이전 단계에서 이미 스케줄 스냅샷으로 계산되어 있어야 한다.

### 5.1 참석 요청 정렬 규칙

- `attendance_requests` 중 `status = 'attend'` 인 요청만 대상이다.
- 정렬 우선순위는 아래와 같다.
1. 전체 시즌 기준 출석률이 낮은 사용자 우선
2. 출석률이 같으면 신청 시각(`requested_at`)이 빠른 사용자 우선

### 5.2 참석자/대기자 분리 규칙

- 정렬된 요청 중 상위 `max_participants` 명을 참석자로 확정한다.
- 나머지는 대기자로 분리한다.
- 현재 기본 최대 인원은 6명이다.

## 6. 대진 생성 공통 규칙

- 생성 결과는 항상 **6경기**여야 한다.
- 각 경기는 `teamA`, `teamB` 두 팀으로 구성된다.
- 각 팀은 2명으로 구성된다.
- 현재 구현은 참석자가 6명 이상이어도 우선 참석자 배열을 기준으로 6경기 로직을 수행한다.
- 현재 구현은 8개 슬롯을 사용하며, 참석자 수가 8명보다 적으면 순환 배치(cyclic fill)로 슬롯을 채운다.
- 생성 단계에서는 저장하지 않고, 마스터가 확정할 때만 실제 경기 데이터로 반영한다.

## 7. 생성 모드별 규칙

현재 지원 모드는 아래 3가지다.

### 7.1 랜덤 모드

- 참석자 목록을 Fisher-Yates 방식으로 무작위 셔플한다.
- 셔플된 순서를 8개 슬롯에 순환 배치한다.
- 아래 6경기 조합 규칙으로 매치를 생성한다.

경기 구성:

1. `g1`: `[0,1] vs [2,3]`
2. `g2`: `[4,5] vs [6,7]`
3. `g3`: `[0,2] vs [4,6]`
4. `g4`: `[1,3] vs [5,7]`
5. `g5`: `[0,3] vs [5,6]`
6. `g6`: `[1,2] vs [4,7]`

### 7.2 실력별 모드

- 참석자 목록을 전체 승률 기준으로 내림차순 정렬한다.
- 게스트는 시즌 통계가 없을 수 있으므로, 비게스트 참석자의 평균 승률을 임시 승률로 사용한다.
- 비게스트가 한 명도 없으면 게스트의 임시 승률은 50으로 본다.
- 정렬 이후 배치 및 6경기 생성 규칙은 랜덤 모드와 동일하다.

### 7.3 혼복 모드

- 여성 그룹은 `gender = 'F' | 'W'` 로 간주한다.
- 남성 그룹은 `gender = 'M'` 으로 간주한다.
- 각 그룹 내부는 먼저 무작위 셔플한다.
- 성비에 따라 아래 분기 규칙을 적용한다.

#### 7.3.1 3M 3F 이상

- 6경기 모두 혼합복식으로 생성한다.
- 여성/남성 인덱스를 회전시키며 조합을 분산한다.

#### 7.3.2 4M 2F 이상

- 4경기는 혼합복식으로 생성한다.
- 2경기는 남성복식으로 생성한다.

#### 7.3.3 2M 4F 이상

- 4경기는 혼합복식으로 생성한다.
- 2경기는 여성복식으로 생성한다.

#### 7.3.4 기타 성비

- 여성/남성 배열을 인터리브한 뒤 6명 기준 순환 배치로 슬롯을 채운다.
- 이후 공통 6경기 조합 규칙을 적용한다.

## 8. 확정 규칙

대진 확정은 아래 조건을 만족할 때만 허용한다.

1. 선택된 일정이 존재해야 한다.
2. 생성된 경기 수가 정확히 6경기여야 한다.

확정 시 동작:

- 기존 `doubles_matches` 중 동일 `schedule_id` 를 가진 경기 데이터는 제거 후 대체한다.
- 새 경기 ID는 `${scheduleId}-m${index}` 형식으로 생성한다.
- 각 경기 데이터는 아래 값으로 저장한다.
  - `schedule_id`: 선택한 일정 ID
  - `date`: 일정 날짜
  - `team_a`, `team_b`: 생성된 팀 구성
  - `score_a`, `score_b`: `null`
  - `result`: `null`
  - `is_confirmed`: `true`

## 9. 결과 입력과의 연결 규칙

- 확정된 대진은 유저 대시보드와 공유 브래킷에서 조회 가능해야 한다.
- 스코어 입력 전까지는 `VS` 상태로 표시한다.
- 스코어 입력 후 `result` 는 아래 규칙으로 계산한다.
  - `scoreA > scoreB` 이면 `teamA`
  - `scoreA < scoreB` 이면 `teamB`
  - 같으면 `draw`

## 10. 권한 규칙

- 대진 생성 및 확정은 마스터만 가능하다.
- 일반 회원은 생성/확정 기능에 접근할 수 없다.
- 일반 회원은 확정된 대진 조회 및 본인 경기 스코어 입력만 가능하다.

## 11. 현재 구현 한계

- 대진 생성 로직이 `MasterPage.tsx` 내부 함수에 위치해 있어 재사용성과 테스트성이 낮다.
- 생성 알고리즘은 현재 6경기 고정이며, 인원 증가에 따른 확장 규칙이 없다.
- 중복 파트너/중복 상대 최소화 규칙이 명시적으로 없다.
- 코트 수, 경기 시간, 휴식 배분, 연속 경기 방지 규칙이 아직 없다.
- 혼복 모드는 성비가 극단적일 때 결과 품질이 운영 기대와 다를 수 있다.

## 12. 향후 확장 요구사항

향후에는 아래 요구사항을 별도 버전 스펙으로 확장할 수 있다.

1. 이전 경기 기록을 반영한 중복 파트너 최소화
2. 상대 조합 중복 최소화
3. 게스트 보정 로직 고도화
4. 코트 수 기반 경기 순서 자동 배정
5. 서버 함수 기반 대진 생성 및 검증
6. 대진 생성 결과 미리보기 비교 기능
