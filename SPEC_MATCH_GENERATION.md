# Tennis Sunlete 대진 생성 스펙

## 1. 문서 목적

- 본 문서는 선레테 서비스의 주간 대진 생성 규칙을 정의한다.
- 대상 기능은 마스터 페이지에서 수행하는 대진 생성 및 확정 기능이다.
- 여성 복식(여복)과 혼합 복식(혼복)의 분리 운영 및 과거 일정에 대한 대진 생성 허용을 포함한다.
- 현재 구현을 기준으로 정리하되, 이후 서버 함수 또는 별도 대진 엔진으로 이전할 수 있도록 정책과 구현 세부를 구분한다.

## 2. 적용 범위

- 대상 페이지: `src/app/pages/MasterPage.tsx`
- 대상 데이터: `schedules`, `attendance_requests`, `doubles_matches`, `users`
- 대상 사용자: 마스터 계정만 대진 생성 및 확정 가능

## 3. 용어 정의

- **참석자(participants)**: 출석 신청자/게스트 중 실제 대진 생성 대상이 되는 사용자 목록
- **대기자(waitlist)**: 참석 신청은 했지만 최대 정원 초과로 대기 상태인 사용자 목록
- **대진 생성(generate draw)**: 메모리/UI 상에서 6경기 구성을 계산하는 행위
- **대진 확정(confirm bracket)**: 생성된 6경기를 `doubles_matches`에 저장하는 행위
- **여복(Women's Doubles)**: 여성 참가자들로만 구성되는 복식 모드
- **혼복(Mixed Doubles)**: 남성/여성 조합의 성비를 고려하여 구성되는 복식 모드

## 4. 생성 대상 일정 조건

대진 생성은 아래 조건을 만족할 때 허용한다.

1. 마스터가 특정 일정을 선택한 상태여야 한다.
2. 일정 상태 조건:
   - `draw_waiting` 상태 일정에서는 언제든 생성 가능
   - 이미 날짜가 지난 일정(과거 기록)에 대해서도 대진 작성 가능
   - 단, 스코어 입력이 완료된 확정 일정은 대진 재생성 불가
3. 참석자 수가 최소 6명 이상이어야 한다.

생성 불가 시 UI는 아래와 같은 수준의 오류를 제공해야 한다.

- 일정 미선택: 경기 일정을 선택하라는 메시지 표시
- 상태 불일치: 일정 마감 후 대진표 생성 대기중 상태에서만 생성 가능 메시지 표시
- 인원 부족: 6경기 생성을 위해 최소 6명의 참석자가 필요하다는 메시지 표시

## 5. 참석자 선정 및 분리 규칙

참석자는 대진 생성 이전 단계에서 이미 스케줄 스냅샷으로 계산되어 있어야 한다.

### 5.1 대진 타입 분리 (신규)

- 대진 생성 시 '여복 대진'과 '혼복 대진'을 선택하여 별도로 생성할 수 있어야 한다.
- 각 타입별로 참여 가능한 성별 및 인원을 필터링하여 대진을 구성한다.

### 5.2 참석 요청 정렬 및 선정

- `attendance_requests` 중 `status = 'attend'` 인 요청을 대상으로 한다.
- 정렬 우선순위:
  1. 출석률이 낮은 사용자 우선
  2. 출석률이 같으면 신청 시각(`requested_at`)이 빠른 사용자 우선
- 상위 `max_participants` 명을 참석자로 확정하고 나머지는 대기자로 분류한다.

## 6. 대진 생성 모드별 세부 규칙

### 6.1 여복 모드 (Women's Doubles)

- 여성 그룹(gender = 'F' | 'W')만을 대상으로 생성한다.
- 기존 실력별 또는 랜덤 모드 로직을 동일하게 적용하되, 여성 참가자 내에서만 슬롯을 배치한다.

### 6.2 혼복 모드 (Mixed Doubles) - 조건 강화

- 남성 그룹(gender = 'M')과 여성 그룹(gender = 'F' | 'W')을 혼합하여 생성한다.
- 혼성 비율 유지 조건:
  - 한 경기에 구성되는 팀 내 혹은 매치 전체의 남녀 성비를 최대한 균등하게 맞추는 로직을 우선한다.
  - 성비 불균형 시 아래 분기를 따른다.
- **3M 3F 이상**: 전 경기 혼합복식 (남녀 파트너 구성 우선)
- **4M 2F / 2M 4F**: 가능한 범위 내에서 혼복을 생성하고, 나머지는 단일 성별 복식으로 배치
- **기타 성비**: 그룹 내 무작위 셔플 후 인터리브(Interleave) 방식을 사용하여 성별이 교차되도록 슬롯을 채운다.

## 7. 대진 생성 및 확정 공통 규칙

- 결과는 기본적으로 **6경기** 구성을 원칙으로 한다.
- 각 경기는 `teamA`, `teamB` 두 팀으로 구성되며, 각 팀은 2명으로 구성된다.
- 참석자 부족 시 순환 배치(cyclic fill)를 통해 8개 슬롯을 채운다.
- 생성 단계에서는 저장하지 않고, 마스터가 확정할 때만 실제 경기 데이터로 반영한다.
- 확정 시 `${scheduleId}-m${index}` 형식으로 저장하며, 기존 데이터가 있을 경우 대체한다.

### 7.1 6경기 조합 규칙

기본 슬롯 배치 후 아래와 같이 경기를 구성한다:

1. `g1`: `[0,1] vs [2,3]`
2. `g2`: `[4,5] vs [6,7]`
3. `g3`: `[0,2] vs [4,6]`
4. `g4`: `[1,3] vs [5,7]`
5. `g5`: `[0,3] vs [5,6]`
6. `g6`: `[1,2] vs [4,7]`

### 7.2 랜덤 및 실력별 모드

- **랜덤**: 참석자 목록을 Fisher-Yates 방식으로 무작위 셔플한 후 슬롯 배치
- **실력별**: 전체 승률 기준 내림차순 정렬하되, 게스트는 비게스트의 평균 승률을 사용

## 8. 권한 및 기타

- 대진 생성 및 확정은 오직 마스터 계정만 가능하다.
- 과거 일정에 대한 대진 수정/생성 시에도 마스터 권한 확인 절차를 거친다.
- 일반 회원은 확정된 대진 조회 및 본인 경기 스코어 입력만 가능하다.

## 10. 결과 입력과의 연결 규칙

- 확정된 대진는 유저 대시보드와 공유 브래킷에서 조회 가능해야 한다.
- 스코어 입력 전까지는 `VS` 상태로 표시한다.
- 스코어 입력 후 `result` 는 아래 규칙으로 계산한다.
  - `scoreA > scoreB` 이면 `teamA`
  - `scoreA < scoreB` 이면 `teamB`
  - 같으면 `draw`

### 10.1 과거 일정 대진 수정 근거

- 과거 일정에 대한 대진 재생성/수정도 개빹 가능하다.
- 단, 진짜 경기 결과가 덱이터베이스에 이미 기록되어 있다면, 대진 교체 및 스코어 변경은 불가하게 가뫍힌다.

## 9. 검증 (신규)

- 각 조건이 맞는지 확인하는 조건 검증 로직을 구현한다.
- 검증 UI를 통해 아래 조건들의 통과 여부를 표시한다:
  - 일정 선택 여부
  - 일정 상태 확인 (draw_waiting 또는 과거 일정)
  - 참석자 수 >= 6명
  - 대진 타입 선택 여부 (여복 또는 혼복)
  - 생성된 경기 수 = 6경기
- 조건 미충족 시 해당 조건에 대한 오류 메시지를 명확히 제시한다.

## 11. 현재 구현 한계

- 대진 생성 로직이 `MasterPage.tsx` 내부 함수에 위치해 있어 재사용성과 테스트성이 낮다.
- 생성 알고리즘은 현재 6경기 고정이며, 인원 증가에 따른 확장 규칙이 없다.
- 중복 파트너/중복 상대 최소화 규칙이 명시적으로 없다.
- 코트 수, 경기 시간, 휴식 배분, 연속 경기 방지 규칙이 아직 없다.
- 혼복 모드는 성비가 극단적일 때 결과 품질이 운영 기대와 다를 수 있다.

## 12. 향후 확장 요구사항

향후에는 아래 요구사항을 별도 버전 스펙으로 확장할 수 있다.

1. 이전 경기 기록을 반영한 중복 파트너 최소화
2. 상대 조합 중복 최소화
3. 게스트 보정 로직 고도화
4. 코트 수 기반 경기 순서 자동 배정
5. 서버 함수 기반 대진 생성 및 검증
6. 대진 생성 결과 미리보기 비교 기능
